# 第四期口播：CC Switch 是什么

> 定位：只讲清概念与关系，不讲安装、导入或具体使用操作。

## 第 1 页｜CC Switch 是啥？

**画面**：一台无人拖拉机停在田间，旁边三个悬浮标签依次出现：Codex、模型、CC Switch。

**口播**：

Codex 装好后，又冒出 CC Switch。它不是新模型，也不是密钥。今天不讲怎么用，就用一个比方把它讲明白：Codex 是一台无人拖拉机；CC Switch，就是它旁边的智能卡槽。先把这三个名字分开，后面就不容易混了。

**翻页衔接**：这台拖拉机，就是 Codex。

## 第 2 页｜一台无人拖拉机

**画面**：无人拖拉机居中，三枚不同颜色的控制芯片依次从侧边滑入，形成“拖拉机 ← 芯片”的关系。

**口播**：

Codex 就是这台能自己干活的无人拖拉机。模型是里面的控制芯片，决定它怎样思考和安排动作。芯片 A 像犁地专家；芯片 B 像播种专家；芯片 C 像精准打药专家。它们不是三台车，而是同一台车能装上的不同大脑。

**翻页衔接**：麻烦在于，以前想换一块大脑，过程很折腾。

## 第 3 页｜以前：拆机焊线

**画面**：维修棚里的拖拉机打开了操作台，散开的线路、工具和设置面板依次出现，画面最终显得复杂拥挤。

**口播**：

没有 CC Switch 时，换一套设置就像把车开进维修棚：拆开操作台，翻旧设置，换上新的，再检查线路。每次都像拆机焊线，步骤多、容易乱，想换也会犹豫。

**翻页衔接**：CC Switch 做的事，就是把这段麻烦路彻底缩短。

## 第 4 页｜现在：智能卡槽

**画面**：拖拉机驾驶座旁出现一个清晰的外置智能卡槽，三张配置卡 A、B、C 随讲述依次柔和地滑入卡槽。

**口播**：

CC Switch 就像驾驶座旁边的智能卡槽。每一套准备好的配置，都是一张卡：卡 A 是一种模型和设置，卡 B 是另一种，卡 C 留给测试。它不造模型，不提供密钥，也不会让模型变强；它只把这些卡整理好，让 Codex 知道当前该接哪一张。

**翻页衔接**：所以，它解决的从来不是能力本身，而是切换的麻烦。

## 第 5 页｜一张图讲透

**画面**：四个对象从左到右逐项点亮：Codex 拖拉机、模型芯片、配置卡、CC Switch 卡槽，最后由一条简洁路径连起来。

**口播**：

最后记住：Codex 是无人拖拉机；模型是控制芯片；配置是一张卡；CC Switch 是管理这些卡的智能卡槽。它把“拆机焊线换脑子”，变成管理不同配置。这样，你就不会把 CC Switch、模型和密钥混成一件事了。

**收尾停顿**：说完最后一句后停一秒，再结束本期视频。
